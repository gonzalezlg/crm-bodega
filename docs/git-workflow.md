# Workflow Git del proyecto CRM Bodega

## Rama principal

main

Nunca desarrollar directamente sobre main.

---

## Flujo de trabajo

1. Crear rama feature.
2. Implementar.
3. Revisar código.
4. Probar.
5. Documentar.
6. Commit.
7. Push de la rama.
8. Crear Pull Request.
9. Revisar PR.
10. Merge a main.
11. Eliminar rama.
12. Actualizar main local.
